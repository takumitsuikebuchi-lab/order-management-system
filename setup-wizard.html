🚚 受注管理システム

初回セットアップウィザード

ようこそ！

受注管理システムのセットアップを開始します。このウィザードでは、システムの使用方法を選択し、必要な設定を行います。

所要時間: 約5〜10分

まず、どのように使用するかを選択してください。

💻 このPCだけで使う（ローカル運用）

インターネット接続不要。データはこのPCのブラウザに保存されます。設定は不要で、すぐに使い始められます。

☁️ 複数のPCで共有する（クラウド運用）

インターネット経由で複数のPCからデータを共有できます。Supabaseという無料サービスを使用します（設定が必要）。

ローカル運用について

ローカル運用では、データはこのPCのブラウザ内に保存されます。

✅ メリット

• 設定不要ですぐに使える
• インターネット接続不要
• 完全にプライベート

⚠️ 注意点

• 他のPCからはアクセスできません
• ブラウザのデータを消すと受注データも消えます
• 定期的にCSV出力でバックアップを推奨

💡 推奨: 週に1回は「CSV出力」ボタンでデータをバックアップしましょう。

クラウド運用について

クラウド運用では、Supabaseという無料サービスを使用してデータを保存します。

✅ メリット

• 複数のPCから同じデータにアクセス可能
• 自動的にクラウドにバックアップ
• PCが壊れてもデータは安全

⚠️ 必要なもの

• インターネット接続
• Supabaseの無料アカウント（メールアドレスのみ）
• 約5分の初期設定

Supabaseプロジェクトの作成

まず、Supabaseで無料アカウントを作成し、プロジェクトを作成します。

  1. Supabase公式サイトを開く
  2. 「Start your project」をクリック
  3. GitHubアカウントでサインイン（または新規作成）
  4. 「New project」をクリック
  5. 以下を入力:
    • Name: order-system（任意の名前でOK）
    • Database Password: 強力なパスワードを設定（メモしておく）
    • Region: Northeast Asia (Tokyo)を選択
  6. 「Create new project」をクリック
  7. プロジェクトの作成完了を待つ（1〜2分）

データベーステーブルの作成

Supabaseプロジェクトにデータを保存するためのテーブルを作成します。

  1. Supabaseの左メニューから「SQL Editor」をクリック
  2. 下のSQLコードをコピーして貼り付け
  3. 右上の「Run」ボタンをクリック
  4. 「Success. No rows returned」と表示されればOK
-- 受注データテーブル作成
create table if not exists public.orders (
  id bigserial primary key,
  order_no text not null unique,
  date date not null,
  customer_name text not null,
  pickup_location text,
  pickup_address text,
  delivery_location text,
  delivery_address text,
  delivery_tel text,
  cargo text not null,
  quantity numeric,
  unit text,
  packaging text,
  unit_price numeric,
  amount_net numeric,
  amount_gross numeric,
  driver text,
  vehicle text,
  instructions text,
  instruction_sheet boolean default false,
  invoice_sent boolean default false,
  payment_received boolean default false,
  order_completed boolean default false,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- アクセス権限の設定
alter table public.orders enable row level security;

create policy "誰でも閲覧可能" on public.orders for select using (true);
create policy "誰でも追加可能" on public.orders for insert with check (true);
create policy "誰でも更新可能" on public.orders for update using (true);
create policy "誰でも削除可能" on public.orders for delete using (true);
